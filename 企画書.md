# ScholarShop 企画書

**― 開封の瞬間まで、誰もパスワードを知らない。校内フリマ×録画連動型解錠システム ―**

---

## 0. 前提

本企画はプログラミングコンテスト出展用の作品であり、校内での限定運用を想定する(一般公開はしない)。
本書はサービスの企画・設計思想をまとめたものであり、技術的な実装詳細には触れない。

---

## 1. 背景と課題

学校には昔から「非公式の物々交換・売買」が存在する。ポテトチップスの貸し借り、過去のレポートやノートの譲渡、卒業する先輩の学ラン・ジャージの引き継ぎ——これらは需要と供給が確かに存在するのに、市場としては何も整備されていない。

現状の校内取引には構造的な問題がある。

| 課題 | 内容 |
|---|---|
| **時間が合わない** | 売り手と買い手が直接会う必要があり、クラス・学年・部活が違うと受け渡しの調整コストが高い |
| **中身の詐称** | 手渡しでも袋の中身までは確認しない。「開けたら空だった/違うものだった」への対抗手段がない |
| **言った言わない** | トラブルが起きても証拠がなく、当事者同士の水掛け論になる |
| **現金トラブル** | 校内での現金授受は紛失・恐喝・カツアゲ的構図など、金額以上のリスクを生む |
| **仲裁者の不在** | 教師に持ち込める性質の取引ではなく、もめごとの裁定者がいない |

つまり校内取引は「**信頼できる第三者が存在しない市場**」である。ScholarShopは、この第三者を人の善意ではなく**システムの構造**で作り出す。

---

## 2. コンセプト

> **「初めて知る」という事実が、そのまま証拠になる。**

フリマ取引最大のリスクは受け渡し時の詐称(空・偽物・破損)である。大手フリマアプリはこれを事後の通報・補償・アカウント評価で緩和しているが、ScholarShopは**開封の瞬間そのものを設計する**ことで、トラブルの発生と同時に証拠が完成している状態を作る。

核となる仕掛けは「**録画連動型解錠システム**」:

1. 出品者は商品を南京錠付きでロッカーに投函する。**錠の番号は出品時にシステムだけに預けられ、購入者はもちろん運営スタッフも取引の瞬間まで見ない**
2. 購入者はロッカーの前で「受け取り開始」ボタンを押す。**スマホの録画が開始され、録画開始より前には決して番号は表示されない**
3. 購入者は**カメラを回しながら、たった今知った番号で**南京錠を開け、中身を確認する
4. 動画は自動で運営に送信・保管される

この設計の妙は、**購入者が開封前に中身を知り得ない/すり替え得ないことを、情報の流れ自体が保証している**点にある。録画に映る「初開封」は演出不可能である。なぜなら、録画開始前の購入者は物理的にロッカーを開けられない(番号を知らない)からだ。

## 3. サービス全体像

### 3.1 登場人物

- **出品者(生徒)** … 商品を出品し、投函する。売上はポイントで受け取り、申請により換金する
- **購入者(生徒)** … ポイントで購入し、ロッカーで録画しながら受け取る
- **運営(生徒スタッフ)** … ポイントのチャージ・換金の窓口、紛争時の動画審査と裁定を行う。**取引そのものには一切立ち会わない**

### 3.2 取引フロー

```
[出品者]                     [システム]                    [購入者]
出品(商品情報+南京錠番号) → 番号を封印保管
                                                      ← 商品を閲覧・購入(ポイント)
                            代金をエスクロー(一時預かり)
商品を投函・施錠 → 「投函済み」に更新 → 購入者に通知
                                                      ← ロッカー前で「受け取り開始」
                            録画開始を確認 → 番号を初開示
                                                      ← 録画しながら開錠・中身確認
                            動画を自動送信・保管
                            ┌ 問題なし → 取引完了、出品者へ入金
                            └ 問題あり → 通報 → 運営が動画審査 → 返金
```

### 3.3 ロッカー運用

- 受け渡し場所は**校内の空きロッカー**を基本とし、出品者の個人ロッカーの利用も認める
- 南京錠は出品者が用意し、**番号は出品時に登録**(以後、画面にも運営にも表示されない)
- 投函のタイミングは出品者の任意(出品と同時でも、売れてからでもよい)。「投函済み」操作をした時点で受け取り可能になる
- ロッカー番号・場所は取引成立後、取引ページのチャットで購入者にのみ伝える
- **番号は取引ごとの使い捨て**とし、取引完了後に出品者は必ず番号を変更する(使い回すと過去の購入者に漏れるため)

### 3.4 ポイント経済

- 校内で使える**独自ポイント制**(1ポイント=1円相当)
- チャージは運営窓口で現金と引き換え。**支払額ぴったりのチャージでも、まとまった額の事前チャージでもよい**
- 購入時、代金ポイントは即座に運営預かり(エスクロー)となり、取引完了まで出品者には渡らない
- 売上ポイントは**申請換金式**。出品者が換金申請すると、運営窓口で現金化される
- 返金はポイント残高への戻しで完結するため、**紛争処理に現金の逆流がない**

## 4. コアアイデア詳解:録画連動型解錠システム

### 4.1 情報の非対称性を「守り」に使う

通常、取引における情報の非対称性は市場を壊す方向に働く(レモン市場)。売り手だけが中身を知っている状態が、詐欺の温床になる。

ScholarShopはこの非対称性を**逆手に取る**。「購入者が番号を知らない」という非対称性を受け取りの瞬間まで人工的に維持し、それを解消するタイミングを録画開始と同期させることで、非対称性の解消イベント(=開封)を必ず記録上に残す。

- 録画開始**前**の購入者:番号を知らない → ロッカーを開けられない → 中身に触れられない
- 録画開始**後**の購入者:番号を知る → しかしその時点からすべてがカメラに映っている

したがって「録画に映る開封」は必ず「正真正銘の初開封」であり、**動画の証拠能力をシステムの構造が担保する**。証人も立ち会いも不要。

### 4.2 インセンティブ設計:録画は強制ではなく自己利益

購入者に録画を「義務」として課すのではない。**録画しなければ番号が手に入らない**だけであり、録画は購入者にとって「自分が返金を受けるための唯一の保険」である。

- 購入者は自己防衛のために**進んで**録画する
- 運営は「ちゃんと録画してくださいね」と善意に頼む必要がない
- 録画せずに(できずに)受け取る手段が存在しないため、「録画していないトラブル」がそもそも発生しない

**性善説の仕組みは破られるが、自己利益と一致した仕組みは破る動機がない。** これが本システムのリスク管理思想である。

### 4.3 双方向の保険であること

見落とされがちだが、この仕組みは**出品者も守る**。

- 購入者が「空だった」と虚偽の通報をしようとしても、動画には実物が映っている
- 「動画がない通報」は制度上不成立(録画なしでは開封できないため)
- つまり出品者は、誠実に投函しさえすれば言いがかりから自動的に守られる

売り手のリスクと買い手のリスクを、**同じ一本の動画**が同時に打ち消す。追加コストゼロの相互保険である。

### 4.4 運営の免責構造

運営は現物を検品しない。それでも成立するのは、裁定材料が常に「動画」という客観物だからである。

- 運営の役割は「動画を見て、返金するか支払うかを決める」ことだけ
- 判断基準が明確(動画に何が映っているか)なので、裁定の属人性・恣意性が排除される
- 「運営が見ていなかったから起きた事故」という類のクレームが構造上発生しない

生徒運営という「弱い運営主体」でも回るのは、**信頼の源泉を運営者の人格からシステムの構造へ移した**からである。

## 5. 悪意シナリオ耐性分析

「どう攻撃されうるか」を先回りして検討する。コンテストにおいても、この分析自体が本作品の見どころである。

| # | 攻撃シナリオ | 結果 |
|---|---|---|
| 1 | 出品者が空・偽物を投函する | 購入者の開封動画が証拠となり返金。出品者は売上を得られない。**騙す動機自体が消える** |
| 2 | 購入者が実物を受け取ったのに「空だった」と主張する | 動画に実物が映っており主張は不成立 |
| 3 | 購入者が録画せずに開封し、後から文句を言う | 不可能。録画開始前に番号を知る手段がない |
| 4 | 購入者が録画を途中で止めて中身をすり替える | 動画が「開錠→中身確認」まで連続していない場合、証拠として不採用=返金不成立。**不完全な動画は撮った本人だけが損をする** |
| 5 | 購入者が番号だけ見て開封せず、後日録画なしで開ける | 番号表示の時点で録画・送信は始まっており、その取引の受取期限を設定すれば時間差攻撃も塞げる |
| 6 | 過去の購入者が使い回された番号で他人の商品を開ける | 番号の取引ごと変更ルールで防止(3.3) |
| 7 | 第三者がロッカーを物理的にこじ開ける | 窃盗であり本システムの守備範囲外。ただし被害時も「開封動画が存在しない取引」として異常を検知でき、校内という閉鎖環境では追跡可能性が高い |
| 8 | 出品者と購入者が共謀して返金詐欺を図る | エスクロー方式のため、返金されると出品者に入金されない。**共謀しても合計利得が増えない**(ゼロサム) |

特筆すべきは #1・#4・#8:このシステムでは**不正の試みが「成功しない」のではなく、「試みる動機が経済的に消える」**。抑止をルールと監視ではなく、利得構造で実現している。

## 6. ビジネスコンテスト的側面

コンテスト作品でありながら、収益と持続性のモデルを備える。

### 6.1 収益モデル

1. **取引手数料** … 売買成立時に代金の数%を運営が徴収(フリマアプリの標準モデル)
2. **ポイントの滞留(フロート)** … 事前チャージ式のため、使われていないチャージ残高と未換金の売上が常に運営側に滞留する。この「先に現金を集め、後で支払う」構造は交通系ICや商品券と同じキャッシュフロー上の強みであり、運営資金の安定化に寄与する
3. **換金の摩擦** … 申請換金式であるため、少額の売上は換金されず次の購入に回りやすい。**ポイントが校内で循環し、外に漏れにくい経済圏**が形成される

### 6.2 市場の特性

- 校内市場は狭いが、**参加者全員が同じ建物に毎日通う**という物流上の奇跡的条件を持つ(配送コスト・受取時間の問題が原理的に存在しない)
- 制服・教材・部活用品など、**校内でしか価値を持たないが校内では確実に需要がある商品**が存在する(学ランの中古市場は当該校の生徒にとってのみ成立する)
- 卒業・進級という**定期的かつ予測可能な大量供給イベント**がある

### 6.3 拡張の展望

- 学校間での相互乗り入れ(近隣校との教材・制服流通)
- 文化祭・バザーでの一時開催モード
- 「開封の瞬間を記録する受け渡し」という仕組み自体は、校外のロッカー受け渡し取引一般に応用可能

---

## 7. 運営体制

- 運営は生徒スタッフ数名(生徒会・委員会活動を想定)
- 業務は①チャージ/換金窓口(昼休み等の定時運用)、②通報時の動画審査、③禁制品等の出品パトロール、の3つのみ
- 取引の受け渡しに立ち会う必要がないため、**取引量が増えても運営の労働量がほぼ増えない**(スケールする)

---

## 8. 成功指標(KPI)

| 指標 | 意味 |
|---|---|
| 取引成立数 / 月 | 市場としての基礎体力 |
| 紛争発生率(通報数 ÷ 取引数) | システムの抑止力の実証。**低いほど「動機を消す設計」が機能している証拠** |
| 紛争解決までの平均時間 | 動画による裁定の効率性 |
| ポイント再利用率(換金せず次の購入に使われた割合) | 経済圏の循環性 |
| リピート出品・購入率 | 信頼の定着 |

---

## 9. まとめ

ScholarShopは「校内フリマ」という身近な題材に、以下の3つの発明を重ねた作品である。

1. **録画連動型解錠** … 「番号を知る瞬間」と「録画の開始」を同期させ、初開封の真正性を情報の流れで保証する
2. **自己利益駆動のリスク管理** … 録画を義務ではなく購入者自身の保険にすることで、善意に依存しない証拠収集を実現する
3. **利得構造による不正抑止** … エスクローと動画裁定の組み合わせにより、不正を「防ぐ」のではなく「割に合わなくする」

信頼できる第三者がいない市場に、人ではなく**構造**を第三者として据える——それがScholarShopの答えである。
